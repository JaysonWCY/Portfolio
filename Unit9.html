1.

Cyclomatic complexity is used to monitor the number of different flows of which a program follows while operating. This is used to measure the independent paths and is considered to portray how complicated or messy a system is.

In terms of object-oriented programming, which focuses on reusability and making complex systems more modular, it is a useful tool. If there are so many paths that a code can follow, then it may be an indicator that the system is not modelled efficiently.

The issues come as object-oriented programming tends to be more on a level of a single system path as opposed to the consideration of the system design as a whole. As an example, a system with many paths and features could still have very modular and easily understood code. It also doesnâ€™t consider other aspects such as inheritance and polymorphism.

Overall, both cyclomatic complexity and object-oriented programming have the same goal of making clear, easy to read, and straightforward code, but both consider different aspects of a system, which does not massively coincide.

Other metrics can be used to measure the complexity of object-oriented designs better, such as Weighted Methods per Class (WMC) or Coupling Between Object Classes (CBO).
