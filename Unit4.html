Exploring Programming Language Concepts

1. What is ReDOS and what part do 'Evil Regex' play?


ReDOS stands for regular expression denial of service. This is the exploitation of regular expressions (Regex) to gain control of a system by consuming huge amounts of the limited computational resource, which in turn prevents service to other users. This is a method used to take hostage of a system or to overall be a nuisance. Evil Regex can be used to cause exponential complexity or backtracking loops which causes a lot of resources to be taken up.


2. What are the common problems associated with the use of regex? How can these be mitigated?


Common issues with regex are that it is very difficult to read as it tends not to be very intuitive. Regex also tend to pose as a problem for security as mentioned before. These can be prevented by sanitising user data and checking user inputs as closely as possible.


3. How and why could regex be used as part of a security solution?


Regex can also be used as a security solution as it is also a tool that can be used to check user inputs to prevent unwanted forms of data.
